# Plan: Check Www-Authenticate Header Before Refreshing Token

## Problem
ApiClient gọi `_refreshToken()` trên MỌI 401 vô điều kiện. Cần chỉ refresh khi 401 thực sự do token (Bearer error).

## Solution

### 1. Thêm helper parse header
```dart
bool _isBearerTokenError(http.Response response) {
  final authHeader = response.headers['www-authenticate'] ?? '';
  return authHeader.toLowerCase().startsWith('bearer ');
}
```

### 2. Sửa `_sendWithRefresh`
```dart
if (authRequired && response.statusCode == 401 &&
    _isBearerTokenError(response) && await _refreshToken()) {
  response = await execute();
}
```

### 3. Sửa 3 inline handlers (postJsonStream, postMultipart, putMultipart)
Áp dụng cùng pattern: thêm `_isBearerTokenError(response)` vào điều kiện.

### 4. `_onSessionExpired` callback
Giữ nguyên — chỉ gọi khi refresh token thất bại (user đã login, token expired, refresh cũng fail).

### Files to modify
- `lib/services/api_client.dart`

### Verification
- `flutter analyze` — no errors
- Invalid token → detects Bearer error → tries refresh → fail → session expired
- Login/register → `authRequired: false` → skips entirely
- Normal flow → preserved
